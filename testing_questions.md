## Test 5

**Question:** What are the main rules or principles of Extreme Programming?

**Source/Page:** Pages 162, 168, 9, 164, 160, 163

**Answer:**  
  Based on the handbook information, Extreme Programming applies four rules in developing a software project:
  **Communication:** The programmer must communicate with the customer to elicit requirements (emphasizing customer satisfaction) and with fellow workers (emphasizing teamwork).     **Simplicity:** The design is maintained as simply as possible.     
  **Feedback:** The software is tested from its early stages, feedback is obtained, and changes are made in a cyclical process.     
  **Courage:** The programmer can make changes even at the last stages and implement new technologies as and when they are introduced.

## Test 7

**Question:** What is software testing and what is its purpose?

**Source/Page:** Pages 244, 253, 251, 247, 252, 249

**Answer:**  
Based on the provided handbook, here is what software testing is and its purpose: *   
**What it is:** Software testing is a critical component of the software development process and a critical element of software quality assurance (SQA). In this process, a software system is executed to determine whether it matches its specification and executes in its intended environment. It focuses on the logical internals of the software (ensuring all statements have been tested) and on the functional externals.     
**Its purpose (and goal):**       * To uncover and correct errors found in software.      * To find differences between the expected behavior specified by system models and the observed behavior of the system once source code is generated.      * To ensure that defined input will produce actual results that agree with required results.


## Test 8

**Question:** What are the different types of software maintenance?

**Source/Page:** Pages 270, 268, 235, 272, 454, 271, 276

**Answer:**  
Based on the provided handbook, the types of software maintenance are categorized in two different ways: 
**According to Page 270, maintenance activities are broken down into four sub-activities:**   
**Corrective maintenance:** Fixing bugs or errors in the system as they are discovered.   
**Adaptive maintenance:** Making a computer program usable in a changed environment.   
**Perfective maintenance or enhancement:** Improving the software's functionality as a result of end-user requests to improve product effectiveness.   
**Preventive maintenance or reengineering:** Performing   premaintenance   in order to prevent system problems. 
**According to Page 235, there are three types of software maintenance:**  
1. **Maintenance to repair software faults** (such as coding errors, design errors, or requirement errors).  
2. **Maintenance to adapt the software to a different operating environment**.  
3. **Maintenance to add to or modify the system’s functionality** (driven by business changes, changing markets, laws, etc.).
